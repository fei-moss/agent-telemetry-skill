# Agent Telemetry 用户侧使用文档

本文档面向在本地运行 Agent（Claude Code、Codex CLI、OpenClaw、Hermes 等）的
用户。启用后，你的 Agent 会把运行过程中的关键记录上报给产品方，用于排查问
题、分析性能、改进 Agent 体验。

## 1. 这是什么

Agent Telemetry 是一个本地运行的观测组件。安装一次后自动工作，记录 Agent
每次运行时发生了什么：

| 类型 | 示例 | 默认是否上传完整内容 |
| --- | --- | --- |
| Agent 运行 | 一次会话、一次后台任务 | 不涉及正文 |
| 决策记录 | Agent 为什么选择某个工具 | 上传结构化摘要 |
| 工具调用 | 调用了搜索、文件、浏览器、数据库等工具 | 结果正文不上传；参数（如命令行、文件路径）经密钥清洗并截断到 500 字符后上传 |
| 模型调用 | 调用了哪个模型、耗时、token 用量、状态 | 不上传完整 prompt 和 output |
| 检索调用 | 查询了哪个知识库、命中了多少文档 | 不上传完整 query |
| 错误与重试 | 工具超时、模型失败、重试次数 | 上传错误类型和摘要 |

默认策略是：**只上传排障所需的结构化元数据，不上传完整 prompt、query、tool
output、模型回答或文件内容。**

## 2. 各 Runtime 的采集方式与离开本机的数据

每条数据都带 `telemetry.collection_layer` 标签，表明它来自哪一层：

| 你的 Runtime | 采集方式 | 标签 | 离开本机的数据（默认） |
| --- | --- | --- | --- |
| Claude Code | hooks（写入 `~/.claude/settings.json`） | `hook` | 会话/工具/耗时元数据，参数脱敏 |
| Codex CLI | 日志监听 + turn 结束触发 | `log_watch` | 工具名、模型名、token 用量、起止时间 |
| OpenClaw | TypeScript 插件 | `plugin` | 会话/工具/模型元数据，内容省略 |
| Hermes | Python 插件 | `plugin` | 会话/API/工具元数据，内容省略 |
| 模型自报 | Agent 自己执行 CLI 命令 | `model_reported` | 决策名称、理由、置信度 |

默认**不会**上传：完整用户问题、完整 prompt/message、完整模型回复、完整工具
返回值（含 stdout/stderr、文件内容等嵌套字段，只保留字符数）、API key /
token / password / cookie 等密钥（这些 key 会被替换为 `[REDACTED]`，常见
密钥格式如 OpenAI/Anthropic/AWS/GitHub/Slack/Google key 与 PEM 私钥块也会
按值匹配清洗）。**命令行里的内联凭证**同样会被清洗：`sshpass -p <密码>`、
`--password/--token/--api-key/--secret <值>`、`mysql -p<密码>`、
`curl -u 用户:密码`、URL 中的 `user:pass@host` 都会被替换为 `[REDACTED]`，
命令的其余部分保留可读。

默认**会**上传的结构化元数据包括：工具参数的标量值（如命令行、文件路径、
URL，经上述密钥清洗并截断到 500 字符）。若不希望上传任何参数值，请关闭遥测
（`AGENT_TELEMETRY_ENABLED=0`）或仅使用本地输出模式（不配置 endpoint）。

## 3. 快速启用

如果你拿到的是产品方分发的安装包（`agent-telemetry-skill-x.y.z.tar.gz` 或
`.zip`），先解压并进入目录（包内附有更简短的 `INSTALL.md` 快速指引）：

```bash
tar xzf agent-telemetry-skill-0.1.0.tar.gz
cd agent-telemetry-skill-0.1.0
```

如果你使用的是源码仓库，则进入仓库根目录。以下命令均在该目录执行：

```bash
python3 scripts/setup.py --auto
```

`setup.py --auto` 会自动检测你安装了哪些 Agent runtime，并安装对应的适配
器，同时初始化配置文件 `~/.agent-telemetry/config.json`。

也可以只为某一个 runtime 单独安装：

```bash
python3 adapters/claude_code/install.py install --yes   # Claude Code
python3 adapters/codex/install.py install --yes         # Codex CLI
python3 adapters/openclaw/install.py install --yes      # OpenClaw
python3 adapters/hermes/install.py install --yes        # Hermes
```

验证组件可以正常工作：

```bash
PYTHONPATH=. python3 -m agent_telemetry_skill.cli status
PYTHONPATH=. python3 -m agent_telemetry_skill.cli demo
```

`status` 会打印当前配置、本地缓存（spool）深度和会话数；`demo` 会在终端输
出几条示例 JSON。

## 4. 连接到产品方上报地址

产品方通常会提供：

- 上报地址：`AGENT_TELEMETRY_ENDPOINT`
- 租户 ID：`AGENT_TELEMETRY_TENANT`
- 接入 token：`AGENT_TELEMETRY_TOKEN`
- 服务名：`AGENT_TELEMETRY_SERVICE`

方式一：写入配置文件 `~/.agent-telemetry/config.json`（建议 `chmod 600`）：

```json
{
  "endpoint": "https://telemetry.example.com/v1/traces",
  "token": "your-ingest-token",
  "service": "my-local-agent",
  "tenant": "your-tenant-id"
}
```

方式二：环境变量（优先级高于配置文件）：

```bash
export AGENT_TELEMETRY_ENDPOINT="https://telemetry.example.com/v1/traces"
export AGENT_TELEMETRY_TOKEN="your-ingest-token"
export AGENT_TELEMETRY_SERVICE="my-local-agent"
export AGENT_TELEMETRY_TENANT="your-tenant-id"
```

发送测试 trace：

```bash
PYTHONPATH=. python3 -m agent_telemetry_skill.cli demo \
  --otlp-endpoint "$AGENT_TELEMETRY_ENDPOINT"
```

如果产品方后台能看到 `agent.run demo`，说明连接成功。

**不配置 endpoint 也可以用**：数据会持久化在本地
`~/.agent-telemetry/spool/`，之后随时可以补传：

```bash
PYTHONPATH=. python3 -m agent_telemetry_skill.cli drain
```

## 5. 本地预览，不上传到产品方

想先看看会产生什么数据，可以启动本地接收服务：

```bash
PYTHONPATH=. python3 server/minimal_ingest.py --port 4318 --output traces.jsonl
```

另开一个终端发送：

```bash
PYTHONPATH=. python3 -m agent_telemetry_skill.cli demo \
  --otlp-endpoint http://127.0.0.1:4318/v1/traces
```

记录会写入当前目录的 `traces.jsonl`。

## 6. 隐私设置

默认不上传完整内容（等价于 `AGENT_TELEMETRY_CAPTURE_CONTENT=0`）。
如果产品方要求开启完整内容采集（上传完整 prompt、query、输出）：

```bash
export AGENT_TELEMETRY_CAPTURE_CONTENT="1"
```

建议只在以下情况下开启，并在调试结束后立即恢复为 `0`：

- 你明确同意上传完整内容
- 正在排查一个具体问题
- 开启时间有限

### 如何确认敏感信息被脱敏

运行 `PYTHONPATH=. python3 -m agent_telemetry_skill.cli demo`，你应该能看到：

```json
"tool.arguments.api_key": "[REDACTED]"
```

prompt/query 正文会显示为：

```json
{"content_omitted": true, "char_count": 5}
```

命令行参数中的内联密码会被替换为占位符、其余部分保留，例如：

```json
"tool.arguments.cmd": "sshpass -p [REDACTED] ssh root@example.com uptime"
```

如果你在输出里看到了不应该上传的敏感内容，请暂停使用并联系产品方。

## 7. 关闭与卸载

临时全局关闭（所有采集入口立即变成静默 no-op）：

```bash
export AGENT_TELEMETRY_ENABLED="0"
```

或停止上传但保留本地记录：

```bash
unset AGENT_TELEMETRY_ENDPOINT
unset AGENT_TELEMETRY_TOKEN
```

彻底卸载（移除所有适配器）：

```bash
python3 scripts/setup.py --uninstall
```

或按 runtime 逐个卸载：

```bash
python3 adapters/claude_code/install.py uninstall --yes
python3 adapters/codex/install.py uninstall --yes
python3 adapters/openclaw/install.py uninstall --yes
python3 adapters/hermes/install.py uninstall --yes
```

卸载脚本只移除自己安装的条目，不会动你的其它配置；首次修改过的配置文件都有
`.bak-agent-telemetry` 备份。最后可删除本地数据目录：

```bash
rm -rf ~/.agent-telemetry
```

## 8. 常见问题

### 我是否必须上传完整 prompt？

不需要。默认不会上传完整 prompt。完整内容采集必须显式开启
`AGENT_TELEMETRY_CAPTURE_CONTENT=1`。

### 上报失败会影响我的 Agent 吗？

不会。所有采集入口都有兜底异常处理，永远不阻塞、不报错给宿主 Agent；上报
失败的数据留在本地 spool，Agent 主流程不受影响。

### 网络断开时会怎样？

不会丢数据。span 会先持久化到磁盘上的 `~/.agent-telemetry/spool/`，**进程退
出后依然保留**。网络恢复后会被自动补传（hook/watcher 会触发 drain），也可以
手动执行 `PYTHONPATH=. python3 -m agent_telemetry_skill.cli drain`。

### 产品方能看到我的 API key 吗？

默认看不到。`api_key`、`token`、`password`、`authorization`、`cookie` 等字段
会被替换为 `[REDACTED]`，字符串里的 `sk-...`、`Bearer ...`、JWT、
AWS/GitHub/Slack/Google key、PEM 私钥块也会被识别替换；命令行里的内联凭证
（`sshpass -p`、`--password`、`curl -u`、URL 里的 `user:pass@` 等）同样会被
清洗。这条防线已在真实数据上验证过。

### 我能只在出问题时开启吗？

可以。建议默认关闭完整内容采集，只保留基础 telemetry；需要排查时再短时间开
启 `AGENT_TELEMETRY_CAPTURE_CONTENT=1`。

### 我如何知道上报是否成功？

运行 `PYTHONPATH=. python3 -m agent_telemetry_skill.cli status`，查看
`spool_depth`：为 0 表示本地数据都已发出。远端确认请让产品方查后台是否出现
你的 `agent.run` trace。

## 9. 排查问题

### 提示找不到模块

请确认你在解压后的安装包目录（或源码仓库根目录）下，且命令带了
`PYTHONPATH=.`：

```bash
cd agent-telemetry-skill-0.1.0   # 或源码仓库目录 agent-telemetry-skill
PYTHONPATH=. python3 -m agent_telemetry_skill.cli demo
```

### 上报地址连接失败

检查：

- `AGENT_TELEMETRY_ENDPOINT` 是否正确（需要包含 `/v1/traces` 路径）
- 网络是否可访问产品方服务
- token 是否过期
- 本地防火墙或代理是否拦截

连接失败期间数据不会丢，留在 spool 里，恢复后 `drain` 即可。

### 后台看不到 trace

```bash
# 1. 本地确认组件正常
PYTHONPATH=. python3 -m agent_telemetry_skill.cli demo

# 2. 查看是否有数据积压、配置是否正确
PYTHONPATH=. python3 -m agent_telemetry_skill.cli status

# 3. 手动补传
PYTHONPATH=. python3 -m agent_telemetry_skill.cli drain
```

## 10. 推荐使用方式

- 平时：`AGENT_TELEMETRY_CAPTURE_CONTENT=0`（默认即是）
- 排障时：临时 `AGENT_TELEMETRY_CAPTURE_CONTENT=1`，结束后立即改回
- 不要把 ingest token、API key、password 粘贴到公开聊天、日志或工单里
- `~/.agent-telemetry/config.json` 建议保持 `0600` 权限

更多细节：架构与失败语义见 [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)，
上报协议见 [docs/PROTOCOL.md](docs/PROTOCOL.md)。
