# Agent Telemetry 使用说明（产品方 / 运维侧）

本文面向**接收遥测数据的产品方**：如何搭建 ingest、发放 token、以及后端必须
实现的去重规则。用户侧启用流程见 [用户侧使用文档.md](用户侧使用文档.md)；
架构与失败语义见 [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)；线上协议的
规范定义见 [docs/PROTOCOL.md](docs/PROTOCOL.md)。

## 1. 你会收到什么

客户端（hook / 插件 / 日志监听 / 模型自报）以 OTLP/HTTP JSON 协议 POST 到你
提供的 `/v1/traces` 端点。span 命名遵守固定约定（`agent.run <name>`、
`execute_tool <tool>`、`chat <model>`、`retrieve <source>`），属性沿用 OTel
GenAI 语义，且每个 span 必带：

- `tenant.id`：客户端声明的租户（**不可信**，见第 3 节）
- `telemetry.collection_layer`：`hook` / `log_watch` / `model_reported` /
  `plugin` / `sdk`，标识数据来源层与可信度
- 客户端已完成默认脱敏（敏感 key → `[REDACTED]`，内容字段 →
  `{"content_omitted": true, "char_count": N}`；已知密钥形态如
  OpenAI/Anthropic/AWS/GitHub/Slack/Google key、JWT、PEM 私钥块，以及命令行
  内联凭证如 `sshpass -p`、`--password/--token/--api-key`、`curl -u`、URL
  `user:pass@` 均按值清洗）

完整字段与编码规则以 [docs/PROTOCOL.md](docs/PROTOCOL.md) 为准。

**时间语义注意**：`log_watch` 层是对本地会话日志的**历史回放**——span 与
event 的时间戳取自日志原始时刻，可能远早于服务端接收时刻（首次安装后会回放
存量日志）。后端的排序、聚合一律以 span 自身的
`startTimeUnixNano`/`endTimeUnixNano` 为准，不要使用接收时间；event 时间戳
保证落在所属 span 的时间窗口内。`chat` span 在 `log_watch` 层可能持续时间为
0（日志只记单个时间点），属预期而非异常。

## 2. 搭建 Ingest

### 开发期：minimal_ingest

仓库自带一个零依赖的本地接收服务，落 JSONL 文件，便于联调：

```bash
PYTHONPATH=. python3 server/minimal_ingest.py --host 127.0.0.1 --port 4318 --output traces.jsonl
```

它接受 `POST /v1/traces`（OTLP JSON）和 `POST /ingest`，每条请求追加写一行
`{"received_at_unix_ms": ..., "path": ..., "payload": ...}`。

验证链路（另开终端）：

```bash
PYTHONPATH=. python3 -m agent_telemetry_skill.cli demo \
  --otlp-endpoint http://127.0.0.1:4318/v1/traces
```

### 生产：OpenTelemetry Collector

最小配置在 `collector/otel-collector.yaml`（OTLP HTTP receiver + batch +
resource processor，debug exporter 占位）：

```bash
docker run --rm -p 4318:4318 \
  -v "$PWD/collector/otel-collector.yaml:/etc/otelcol/config.yaml" \
  otel/opentelemetry-collector-contrib:latest \
  --config /etc/otelcol/config.yaml
```

生产部署建议在 Collector / ingest 层开启：

- retry + sending queue + persistent file storage（客户端是 at-least-once，
  服务端也要抗抖动）
- 鉴权层校验 `Authorization: Bearer <token>`（Collector 前置网关或
  auth extension）
- resource processor 按 token 注入真实 tenant/workspace（覆盖客户端声明值）
- tail sampling（按需）
- secrets scrubber / 二次脱敏兜底（客户端默认已脱敏，但不要依赖它）

下游可转发到 Langfuse、Phoenix、Tempo、ClickHouse 或自研 trace store。

## 3. Token 发放指引

- **一租户一 token**（或一租户多 token 便于轮换），token 只授予"写入
  trace"单一能力，不可用于查询或管理。
- **租户归属以服务端为准**：必须从 token 反查租户，覆盖/校验客户端上报的
  `tenant.id`，防止本地配置伪造跨租户写入。
- 给用户的接入包只需四个值：`AGENT_TELEMETRY_ENDPOINT`、
  `AGENT_TELEMETRY_TOKEN`、`AGENT_TELEMETRY_TENANT`、
  `AGENT_TELEMETRY_SERVICE`（写入环境变量或
  `~/.agent-telemetry/config.json`，提醒用户 `chmod 600`）。
- 支持服务端吊销与定期轮换；token 疑似泄露时立即吊销（客户端无须改代码，
  换值即可）。
- 限流按 token 维度做；4xx/5xx 都不会影响用户的 agent（数据会留在客户端
  spool 重试），所以可以放心拒绝异常流量。

## 4. 后端去重规则（必须实现）

客户端交付语义是 **at-least-once**，且同一动作可能被多个采集层同时捕获：

1. **按 `span_id` 精确去重**：重试投递的 span 其 `trace_id`/`span_id` 完全
   相同，幂等写入即可（如 ClickHouse ReplacingMergeTree、唯一键 upsert）。
2. **跨层语义去重**：同一 `session.id` 下同一动作出现多个来源时，按
   `telemetry.collection_layer` 可信度取优：
   `hook` / `plugin` / `sdk` ＞ `log_watch` ＞ `model_reported`。
   `log_watch` span 带 `telemetry.source.file` 可辅助溯源与匹配。
3. **`model_reported` 仅作补充**：decision/rationale 等是其它层拿不到的独有
   数据，照常入库；但不要用它修正其它层的计时、状态或用量统计。

## 5. 打包分发给用户

发给用户的安装包用打包脚本生成（自动排除 tests / 缓存 / 构建产物，并附带
中文 `INSTALL.md` 快速指引）：

```bash
python3 scripts/build_package.py            # 产物输出到 dist/
```

输出 `dist/agent-telemetry-skill-<版本>.tar.gz` 与 `.zip` 各一份（版本号取自
`pyproject.toml`），并打印各自的 sha256——发布时把校验值一并提供给用户。

分发建议：

- 升级版本时先改 `pyproject.toml` 的 `version` 再打包。
- 随包发给用户的接入信息只需四个值（见第 3 节）；token 不要写进包里。
- 用户侧的完整启用文档即包内 `用户侧使用文档.md`，安装入口是
  `python3 scripts/setup.py --auto`。

## 6. 联调检查清单

```bash
# 客户端侧
PYTHONPATH=. python3 -m agent_telemetry_skill.cli status   # 配置 + spool 深度
PYTHONPATH=. python3 -m agent_telemetry_skill.cli demo --otlp-endpoint <你的端点>
PYTHONPATH=. python3 -m agent_telemetry_skill.cli drain    # 手动补传积压

# 全量测试
PYTHONPATH=. python3 -m unittest discover -s tests
```

后台应看到 `agent.run demo` 根 span 及其 `execute_tool search`、
`chat gpt-5-mini` 子 span，且 demo 数据中 `tool.arguments.api_key` 为
`[REDACTED]`、prompt 内容为 `content_omitted` 形态——如果看到了明文密钥或
完整内容，说明客户端被改动过或开启了 `capture_content`，应当告警。
